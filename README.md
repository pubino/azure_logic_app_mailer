# Azure Logic App Mailer

A secure Drupal 9/10 module that routes outbound emails through an Azure Logic App using Azure Managed Identity for authentication. 

## Features

- **Azure Managed Identity Authentication:** Automatically acquires OAuth2 tokens from Azure's Instance Metadata Service (IMDS) or App Service identity endpoints to authenticate requests securely without storing static credentials.
- **Secure Email Delivery:** Posts email payloads to an Azure Logic App HTTP trigger over HTTPS.
- **Local Development Fallback:** Provides an offline developer fallback that writes out emails to a local directory (`temporary://drupal-mails/`) and logs them to the Drupal watchdog, avoiding accidental production emails during development.
- **Testing Interface:** Includes a built-in admin form to send test emails.
- **Unit Testing:** Comprehensive test suite coverage for both mock environment fallbacks and simulated production environments.

---

## Requirements

- **Drupal** `^9` or `^10`
- **Mail System** (`mailsystem`) module

---

## Installation

1. Place the module in your Drupal site's custom module directory (typically `web/modules/custom/azure_logic_app_mailer`).
2. Enable the module using Drush or the Drupal Admin UI:
   ```bash
   drush en azure_logic_app_mailer
   ```

## Configuration

### 1. Mail System Integration

After enabling the module, configure Drupal to use the Azure Logic App Mailer:

1. Navigate to **Configuration** > **System** > **Mail System** (`/admin/config/system/mailsystem`).
2. Under **Formatter** and **Sender**, select **Azure Logic App Mailer** (Plugin ID: `logic_app_mailer`).
3. Save configuration.

### 2. Environment Variables

The mailer is configured via environment variables to keep sensitive configuration separate from code.

| Variable | Description | Example / Value |
| --- | --- | --- |
| `AZURE_LOGIC_APP_MAIL_URL` | The HTTP POST trigger endpoint of your Azure Logic App. | `https://prod-XX.eastus.logic.azure.com:443/...` |
| `IS_LOCAL_DEV` | If set to `true`, the mailer runs in offline fallback mode. | `true` or `false` |
| `IDENTITY_ENDPOINT` | *(Optional)* The Azure Managed Identity endpoint URL (automatically set by Azure App Service/Functions/Containers). | `http://127.0.0.1:8081/msi/token` |
| `IDENTITY_HEADER` | *(Optional)* The Azure Managed Identity header secret (automatically set by Azure App Service). | *(Random string generated by Azure)* |

#### Local Development Mode
When `IS_LOCAL_DEV` is `true` or `AZURE_LOGIC_APP_MAIL_URL` is empty, emails are written to:
`temporary://drupal-mails/mail-<timestamp>-<hash>.txt`

And logged to Watchdog with the channel name `mail_fallback`.

#### Managed Identity Token Retrieval Flow
In production (`IS_LOCAL_DEV` is `false`), the module retrieves an access token for `https://management.azure.com/`:
1. If `IDENTITY_ENDPOINT` and `IDENTITY_HEADER` are present (e.g. on Azure Web Apps or Azure Container Apps), it uses those along with the `X-Identity-Header` header.
2. Otherwise, it falls back to the standard Azure VM Instance Metadata Service (IMDS) endpoint (`http://169.254.169.254/metadata/identity/oauth2/token?api-version=2018-02-01&resource=https://management.azure.com/`) with `Metadata: true` header.

## Payload Format

The HTTP POST request sent to the Logic App has the following JSON structure:

```json
{
  "to": "recipient@example.com",
  "subject": "Email Subject",
  "body": "Plain text or HTML email body contents."
}
```

## Verification

To verify that the mailer is functioning correctly, you can send a test email:

1. Navigate to `/admin/config/system/azure-logic-app-mailer/test` (requires the `administer site configuration` permission).
2. Enter a recipient email address.
3. Click **Send Test Email**.
4. Check the Drupal watchdog logs (`/admin/reports/dblog`) or your Azure Logic App execution history to confirm successful delivery.

---

## Running Tests

Unit tests are included to cover local file output, successful production sends with mocked cURL token exchange, and production API failures. Run the tests using PHPUnit:

```bash
vendor/bin/phpunit -c web/core web/modules/custom/azure_logic_app_mailer/tests
```
